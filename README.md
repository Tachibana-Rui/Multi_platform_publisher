# Content Hub

一个本地优先的个人内容中台，用于统一保存原创文案、标签、图片和视频，为后续的平台内容适配与发布流程提供可靠的数据源。

## 当前能力

- 创建、编辑、搜索、筛选和删除内容
- 草稿、就绪、归档状态管理
- 标签归一化与内容来源记录
- 批量拖拽上传图片和视频
- 从小红书公开作品链接导入标题、正文、标签及全部媒体
- 按 Coser、角色和拍摄日期索引、检索本地原始素材
- 使用角色标签、pHash 与 SSIM 自动匹配高清原图
- 使用豆包 Seed 2.0 Lite 生成跨平台标题和文案
- 保存抖音、小红书、B站、快手和视频号独立发布草稿
- 通过可见浏览器发布到抖音、小红书和 B 站，并保存任务状态与作品链接
- 发布时选择公开可见、仅互关好友可见或仅自己可见
- 检测三平台本机登录状态，并从账号管理窗口打开网页登录
- 图片尺寸、文件大小、SHA-256 校验值自动提取
- 通过 FFprobe 提取视频尺寸和时长
- 卡片/列表双视图及响应式界面
- SQLite 本地持久化及素材文件管理

## 启动

当前电脑已经安装了运行依赖，可以直接双击：

```text
start_content_hub.bat
```

也可以从 PowerShell 启动：

```powershell
python run.py
```

应用会自动打开 <http://127.0.0.1:8000>。关闭终端窗口即可停止服务。

首次在其他电脑运行时，先安装依赖：

```powershell
python -m pip install -r requirements.txt
python run.py
```

不希望自动打开浏览器时：

```powershell
python run.py --no-browser
```

## 数据位置

```text
data/content_hub.db       SQLite 数据库
storage/uploads/{内容ID}/  默认素材目录；可从首页“存储设置”修改
```

备份时停止应用，然后一起复制 `data` 和“存储设置”中显示的素材目录。不要只备份数据库，否则内容记录仍在，但对应素材会丢失。

更改素材目录时，应用会先复制并校验已有文件，成功后再切换动态素材路由。旧目录不会自动删除，可在确认新目录完整后自行备份或清理。

## 技术结构

```text
浏览器界面（原生 HTML / CSS / JavaScript）
                    ↓
             FastAPI REST API
                    ↓
      SQLAlchemy + SQLite / 本地素材目录
```

前后端由同一个 Python 进程提供服务，不需要安装 Node.js。SQLite 已启用外键、WAL 和 5 秒忙等待。API 文档可在应用运行时访问 <http://127.0.0.1:8000/docs>。

## 小红书链接导入

在首页点击“链接导入”，粘贴公开作品链接或包含链接的分享文本。系统会：

1. 解析 `xiaohongshu.com` 作品地址或 `xhslink.com` 分享短链。
2. 保存作品标题、正文和话题标签。
3. 下载图文作品的全部图片；视频作品会下载视频及页面提供的封面图。
4. 将媒体统一存入 `storage/uploads/{内容ID}/`，再创建 Content Hub 草稿。

只导入本人原创或已经获得授权的内容。当前实现不会绕过登录、验证码或平台访问限制；仅登录可见、已删除或触发平台验证的作品会返回明确错误。评论、点赞等互动数据不属于作品原始内容，不在本阶段导入范围内。

## 原始素材检索与图片匹配

首页点击“素材检索”，再点击“选择并添加目录”。应用会打开 Windows 原生文件夹选择器；选择完成后立即登记目录，并在后台递归建立图片索引。扫描期间可以继续使用 Content Hub，左侧目录卡片会显示实时进度。

应用会解析以下结构化目录名：

```text
CoserName + CharacterName + YYYYMMDD
Alice+艾德+20260621
```

`+`、全角加号、下划线、横线和空格均可作为分隔符。完成扫描后，可以按 Coser、角色和拍摄日期检索目录。扫描是增量的：文件大小和修改时间没有变化时不会重新计算图片特征；同一个目录不会并行扫描；移除索引也不会删除原始文件。

对于已经导入 Content Hub 的小红书作品，在内容编辑器中点击“开始匹配”：

1. 将 `艾德cos`、`艾德cosplay` 等标签标准化为 `艾德`。
2. 只检索角色名为“艾德”的结构化素材目录。
3. 使用宽高比和 pHash 召回候选图片。
4. 将高清原图缩小到下载图尺寸，通过 SSIM 验证压缩前后是否为同一张图。
5. 高置信度结果自动复制到 `storage/uploads/{内容ID}/originals/`；中等置信度结果等待人工确认。

当前匹配器针对“仅缩放和重新压缩、没有裁剪与文字滤镜”的场景设计。原始素材库始终只读，应用不会移动或修改其中的文件。

## 豆包文案适配

首页点击“AI 配置”，填写火山方舟 API Key，并确认模型或推理接入点 ID。默认配置为：

```text
API：https://ark.cn-beijing.volces.com/api/v3
模型：doubao-seed-2-0-lite-260428
```

API Key 使用当前 Windows 用户的 DPAPI 加密后保存在本机，读取配置的接口只返回配置状态和末四位提示，不会返回完整密钥。

打开任意内容的编辑窗口，点击“平台发布窗口”：

1. 首次打开某个平台时，Content Hub 已有的标题和正文会原样复制，不调用 LLM；无标题内容保持空白，等待用户决定是否生成。
2. 可以在右侧自行选择发布图片；前 4 张图片作为豆包的视觉输入。
3. 默认 Prompt 根据内容标签、目标平台和图片数量生成，例如 `生成 NIKKE 艾德cos 抖音 标题和文案，带有4张图片。`。
4. Prompt、标题和正文都允许人工修改。
5. 点击“一键生成”后会保存豆包结果，按钮变为“再次生成”；继续点击可获得新版本。
6. 每个平台维护独立草稿，不会覆盖 Content Hub 中的原始标题和正文。

## 发布 Agent

平台发布窗口支持抖音、小红书和 B 站。选择素材并保存文案后，点击“准备发布”：

1. Agent 使用独立的 Edge/Chrome 本地配置打开平台创作中心；首次使用时由用户手动扫码或登录。
2. Agent 校验素材，将选中的图片或视频上传，并自动填写标题与正文。
3. 上传完成后任务停在“等待最终确认”。请在平台窗口检查封面、分区、声明、可见范围等平台必填项。
4. 回到 Content Hub 点击“确认并发布”，Agent 才会点击平台发布按钮并记录结果。

首页右上角“账号管理”会检查 Content Hub 专用浏览器中的抖音、小红书和 B 站登录状态。未登录时点击对应平台按钮，在打开的网页中完成登录即可。平台页面若未暴露所选可见范围，发布任务会在人工确认阶段明确提示用户手动核对，避免静默使用错误范围。

浏览器登录状态仅保存在 `data/browser_profiles/{平台}/`，应用不读取账号密码、不导出 Cookie，也不处理验证码。每个平台同一时间只打开一个发布窗口；失败任务可以重试。发布任务会保存创建时的文案和素材快照，后续编辑草稿不会悄悄改变正在进行的任务。

## 开发与测试

```powershell
python -m pip install -r requirements-dev.txt
python -m pytest -q
```

主要目录：

```text
app/main.py              API 和应用入口
app/models.py            SQLite 数据模型
app/assets.py            素材保存与媒体信息提取
app/content_matcher.py   原始素材索引、检索与图片匹配 Agent
app/doubao.py            豆包多模态文案生成客户端
app/platform_adapter.py  平台版本与图片选择
app/publish_agent.py      发布任务编排、状态与审计日志
app/publishers/           抖音、小红书、B站浏览器发布器
app/llm_settings.py      本机 API Key 加密配置
app/static/              Content Hub 前端
tests/test_api.py         API 集成测试
```

## 当前边界

发布 Agent 保留人工登录和最终确认，不绕过验证码或平台风控。平台创作页面改版后，浏览器选择器可能需要同步维护；失败原因会保存在发布任务中。
